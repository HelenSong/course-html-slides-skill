# Markdown 设计稿格式规范

本文档定义了 HTML Slides 设计稿的标准 Markdown 格式。设计稿是从课程大纲到 HTML 文件的中间产物——它用结构化的方式描述每一页的类型、内容、布局和交互，让 HTML 生成有据可依。

---

## 文档结构

```markdown
# [课程/演示名称] — HTML 课件页面设计稿

## 元信息
- 课程名称: [名称]
- 总页数: [N]
- 风格方案: [预设名 或 自定义]
- 目标场景: [课堂投影 / 远程分享 / 自学浏览]

## 页面目录

| 页码 | 文件名 | 类型 | 标题 | 所属部分 |
|---|---|---|---|---|
| P01 | p01-cover.html | 封面 | 课程标题 | — |
| P02 | p02-intro.html | 内容页 | 课程介绍 | 导入 |
| ... | ... | ... | ... | ... |

---

## 逐页设计

### P01 — 封面
- **类型**: 🎬 封面 (cover)
- **背景**: [暗色 / 渐变 / 图片URL]
- **内容**:
  - 眉标: [文字]
  - 主标题: [文字]（强调词: [渐变色词]）
  - 副标题: [文字]
  - 标签: [标签1] [标签2] [标签3]
- **交互**: 无（纯展示）
- **备注**: [可选，给开发者的提示]

### P02 — 课程介绍
- **类型**: 📖 内容页 (content)
- **背景**: 默认底色
- **布局**: 双列（左 1:1.6 右）
- **内容**:
  - 左列: 
    - 标题: [文字]
    - 段落: [正文内容]
  - 右列:
    - 卡片组:
      - 卡片1: [emoji] [标题] — [说明]
      - 卡片2: [emoji] [标题] — [说明]
- **交互**: reveal 入场动画（卡片逐个出现）
- **备注**: 无

### P05 — 第一部分标题
- **类型**: 🔷 幕间标题 (section-title)
- **背景**: 主色渐变
- **内容**:
  - 部分编号: 第 1 部分
  - Emoji: 🌍
  - 标题: [大标题文字]
  - 关键词: [词1] [词2] [词3]
- **交互**: 装饰 blob 浮动动画
```

## 页面类型速查

在设计稿中标注类型时，使用以下标准名称：

| 类型标识 | Emoji | 典型用途 |
|---|---|---|
| `cover` | 🎬 | 课程封面、结尾页 |
| `section-title` | 🔷 | 章节/部分间隔页 |
| `content` | 📖 | 知识讲解、概念介绍 |
| `interactive-demo` | 🤖 | 嵌入演示、工具体验 |
| `task-guide` | 📝 | 操作步骤指引 |
| `practice` | ⏱ | 动手练习（含计时器） |
| `summary` | 📊 | 要点回顾、总结 |
| `qrcode` | 🔗 | 扫码加群、资源链接 |

## 内容描述规范

### 文本内容
- 直接书写，不需要占位符
- 需要强调的词用 `**加粗**` 标注
- 需要渐变色高亮的词用 `==高亮==` 标注

### 布局描述
- 单列: `布局: 单列居中`
- 双列等宽: `布局: 双列（1:1）`
- 左窄右宽: `布局: 双列（左 1:1.6 右）`
- 网格: `布局: 2×2 网格` 或 `布局: 3列网格`

### 交互描述
- `reveal 入场`: 元素按延迟逐个出现
- `click-to-reveal`: 点击逐步显示（参考 click-to-reveal-pattern.md）
- `翻转卡片`: 点击翻转显示背面内容
- `聊天动画`: 分幕打字机对话（参考 chat-animation-pattern.md）
- `iframe 嵌入`: 嵌入外部网页（附 URL 和后备方案）
- `倒计时器`: 练习计时（附时长，如 5min）
- `复制按钮`: 一键复制文本到剪贴板

### 背景描述
- `默认底色` → 使用 `--base-bg`
- `主色渐变` → 使用 `--primary` 渐变
- `辅色渐变` → 使用 `--secondary` 渐变
- `暗色` → 深色背景 + 浅色文字
- `图片: [URL或描述]` → 背景图 + 蒙版

## 设计稿编写原则

1. **一页一段** — 每个 `### PXX` 描述一页，不要在一段中描述多页
2. **内容不超载** — 每页内容量以 1280×800 满屏为限，宁拆勿挤
3. **类型先行** — 先确定页面类型，再填充内容，类型决定了布局骨架
4. **交互明确** — 每个交互元素都要注明触发方式和效果
5. **导师视角** — 在「备注」中记录教学节奏提示（如「此处暂停 2 分钟讨论」）
